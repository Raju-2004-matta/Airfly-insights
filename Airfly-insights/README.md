# ✈️ AirFly Insights — Milestone 1: Data Preprocessing Pipeline

## 📌 Project Overview

**AirFly Insights** is a flight delay analysis project designed to explore airline performance, delay patterns, and operational efficiency.
Milestone 1 focuses on building a **clean, optimized, and reproducible preprocessing pipeline** that transforms raw flight data into an analysis-ready dataset.

The goal of this milestone is to prepare a reliable data foundation that will support Exploratory Data Analysis (EDA), visual dashboards, and predictive modeling in later stages.

---

## 🎯 Milestone 1 Objectives

* Load and explore the raw flight dataset.
* Optimize memory usage for large-scale data processing.
* Perform feature engineering to enhance analytical insights.
* Clean and standardize the dataset.
* Build an automated preprocessing pipeline.
* Generate a processed dataset ready for analysis.

## ⚙️ Preprocessing Pipeline

The pipeline is implemented in:

```
src/preprocessing.py
```

### 🔹 1. Data Loading

* Reads raw flight dataset using pandas.
* Validates schema and column types.

### 🔹 2. Memory Optimization

* Downcasts numeric columns:

  * `int64 → smaller integers`
  * `float64 → float32`
* Reduces memory usage significantly.

### 🔹 3. Feature Engineering

New analytical features created:

* **flight_date** — Combined `year`, `month`, `day`.
* **day_of_week** — Extracted from flight_date.
* **route** — Origin–Destination pair (e.g., JFK-LAX).
* **cancelled** — Flag if departure time is missing.
* **on_time** — KPI where `arr_delay ≤ 0`.

### 🔹 4. Data Cleaning

* Removed rows with missing `arr_delay`.
* Standardized categorical fields.
* Ensured data consistency.

### 🔹 5. Output Generation

* Saved processed dataset automatically into `data/processed/`.

Add the raw dataset into:

```

```
